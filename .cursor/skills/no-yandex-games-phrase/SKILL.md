# Запрет словосочетания «Яндекс Игры»

## Главное правило

При разработке игры **ни в коем случае нигде не писать** словосочетание **«Яндекс Игры»**.

Это касается всего, что относится к проекту игры:

- тексты сюжета и диалогов (`story/`, JSON, скрипты сборки);
- UI и подписи в интерфейсе (`index.html`, `js/ui.js`, `css/`);
- комментарии и JSDoc в коде игры;
- README и любые markdown-файлы в репозитории игры;
- промо-тексты, описания, подсказки для игрока;
- строковые литералы, `content` в CSS, `aria-label`, `title`, alt-тексты.

## Что делать вместо этого

- Не называть платформу в игровом контенте — игроку это не нужно.
- В технических комментариях: «SDK платформы», «игровой SDK», «YaGames SDK».
- В обсуждении с пользователем можно говорить о платформе своими словами, но **не добавлять** фразу «Яндекс Игры» в файлы проекта.

## Перед завершением задачи

1. Проверить изменённые файлы поиском по `Яндекс Игр` и `Яндекс Игры`.
2. Если фраза уже была в затронутом файле — убрать или заменить нейтральной формулировкой.
3. Не вводить фразу в новых коммитах, даже в комментариях «для ясности».

## Примеры

| Нельзя | Можно |
|--------|-------|
| `<!-- SDK Яндекс Игр -->` | `<!-- SDK платформы -->` |
| `Игра для Яндекс Игр` | `Браузерная визуальная новелла` |
| `лидерборд в консоли Яндекс Игр` | `лидерборд в консоли разработчика` |
| `требования Яндекс Игр` (в README игры) | `требования платформы` / `требования модерации` |

## Связанные skills

Технические требования SDK и модерации — в `yandex-games-requirements`, но формулировки для **файлов игры** всё равно без словосочетания «Яндекс Игры».
